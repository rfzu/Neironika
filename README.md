# 🧠 LLM LoRA Fine-Tuning Project

Проект для **тонкой настройки (fine-tuning)** языковых моделей с использованием технологии **LoRA** (Low-Rank Adaptation). Позволяет адаптировать модель под специфические задачи, такие как психологические консультации в стиле Вероники Степановой.

---

## 📖 О проекте

Этот репозиторий содержит инструменты для:
- 📚 Подготовки датасетов в формате JSONL
- 🎯 Обучения LoRA-адаптеров с использованием 4-битной квантизации
- 💬 Запуска чата с настроенной моделью

**Базовая модель:** [`huihui-ai/Llama-3.2-3B-Instruct-abliterated`](https://huggingface.co/huihui-ai/Llama-3.2-3B-Instruct-abliterated)

**Специализация:** Системно-векторная психология, психологические разборы в яркой и провокационной манере.

## 🚀 Быстрый старт

### 1️⃣ Обучение адаптера

```bash
python train.py
```

**Параметры обучения (в `train.py`):**
| Параметр | Значение | Описание |
|----------|----------|----------|
| `r` | 32 | Ранг LoRA матриц |
| `lora_alpha` | 64 | Коэффициент масштабирования |
| `target_modules` | q_proj, k_proj, v_proj, o_proj | Модели для адаптации |
| `epochs` | 3 | Количество эпох обучения |
| `batch_size` | 4 | Размер батча |
| `learning_rate` | 1e-4 | Скорость обучения |

**Особенности:**
- ✅ 4-битная квантизация (BitsAndBytes)
- ✅ Автоматическое сохранение чекпоинтов
- ✅ Подробное логирование в `train.log`
- ✅ Мониторинг VRAM и скорости обучения

---

### 2️⃣ Запуск чата

```bash
python lora_chat.py
```

**Команды в чате:**
- `exit` / `quit` / `q` — выход
- `clear` — очистить историю диалога

**Системный промпт:** Модель выступает в роли психолога Нейроники Степановой, эксперта по системно-векторной психологии.

---

### 3️⃣ Тест базовой модели

```bash
python start_model.py
```

Проверка работы модели без адаптера.

## 🛠️ Подготовка датасета

1. Создайте файлы в формате JSONL:
```jsonl
{"text": "<|begin_of_text|><|start_header_id|>user<|end_header_id|>\n\nВопрос?<|eot_id|><|start_header_id|>assistant<|end_header_id|>\n\nОтвет!<|eot_id|>"}
```

2. Объедините файлы (опционально):
```bash
python merge_jsonl.py
```

3. Отформатируйте датасет:
```bash
python format_dataset.py
```

## 🔧 Конфигурация LoRA

```python
LoraConfig(
    r=32,                    # Ранг матриц
    lora_alpha=64,           # Alpha scaling
    target_modules=[         # Слои для адаптации
        "q_proj", "k_proj", 
        "v_proj", "o_proj"
    ],
    lora_dropout=0.05,       # Dropout
    bias="none",
    task_type="CAUSAL_LM"
)
```

## 📝 Логи

Все логи сохраняются в `train.log` с информацией:
- Потеря (loss) по шагам
- Скорость обучения (learning rate)
- Норма градиента (grad_norm)
- Потребление VRAM
- Скорость обучения (steps/min)

## ⚠️ Важные замечания

1. **CUDA обязателен** — обучение на CPU не поддерживается
2. **Драйверы NVIDIA** — убедитесь, что установлены актуальные драйверы
3. **Датасет** — файл `merged_dataset_formatted.jsonl` должен существовать перед обучением
4. **Память GPU** — при нехватке памяти уменьшите `per_device_train_batch_size`
